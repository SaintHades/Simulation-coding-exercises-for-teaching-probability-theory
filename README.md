# Coding Exercises for Probability — Experiment Repo

This repository contains the code and instructions needed to **reproduce** our data processing, exploratory analyses, intake form visualizations, and statistical modeling for a small-scale, voluntary study on probability learning.

## 0) Quick Start

```bash
# 1) clone
git clone https://github.com/SaintHades/Simulation-coding-exercises-for-teaching-probability-theory.git
cd Simulation-coding-exercises-for-teaching-probability-theory

# 2) create environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install jupyter numpy pandas matplotlib statsmodels

# 3) run notebooks (top-to-bottom)
jupyter lab
# open:
#   exploratory_data_analysis.ipynb
#   intake_form_visualization.ipynb
#   statistic_analysis.ipynb
````

## 2) Data Layout

All source data lives under `data/` with the following structure:

```
data/
├── raw/                        # original exports for all participants
│   ├── basic.csv                   # group/section assignment and basic grouping info
│   ├── final_assessment.csv        # detailed final assessment records
│   ├── grades.csv                  # aggregated coding/handwritten/final scores
│   └── intake_form.csv             # intake form responses and background information
└── cleaned/                    # cleaned data restricted to completed participants
    ├── basic.csv                   # cleaned version of raw/basic.csv
    ├── final_assessment.csv        # cleaned version of raw/final_assessment.csv
    ├── grades.csv                  # cleaned version of raw/grades.csv
    ├── intake_form.csv             # cleaned version of raw/intake_form.csv
    └── status.csv                  # completion status used for participant filtering
```

### 2.1 `data/raw/` (do not edit)

`raw/` contains the original exports for **all participants**, including those who did not finish.

Files:

* `basic.csv`
  Group/section assignment and basic grouping information.
* `final_assessment.csv`
  Detailed final assessment records, including per-subquestion scores and confidence levels.
* `grades.csv`
  Aggregated scores for coding, handwritten, and final assessments.
* `intake_form.csv`
  Intake form responses (e.g., courses taken and background information).

### 2.2 `data/cleaned/` (generated; completed-only)

`cleaned/` contains cleaned versions of the same four files **filtered to only include students who completed the required tasks**, plus an additional status file.

Files:

* `basic.csv`, `final_assessment.csv`, `grades.csv`, `intake_form.csv`
  Same schemas as `raw/`, but restricted to “completed” participants (see `status.csv`).
* `status.csv`
  Records each participant’s section/group and completion status (used to filter completed participants).

### 2.3 Confidence-adjusted final score (what it means)

In `cleaned/grades.csv`, the confidence-adjusted final score is computed using per-subquestion information from `final_assessment.csv`.

Conceptually:

* For each subquestion `q`: `adjusted_q = score_q * confidence_q`
* The overall adjusted score is then aggregated across subquestions (weighted mean as implemented).

> The exact aggregation rule used is implemented in the pipeline code/notebook and should be treated as the source of truth.

### 2.4 `outputs/`

Saved figures and tables generated by notebooks are stored under `outputs/`:

```text
outputs/
├── exploratory_data_analysis/
├── intake_form_visualization/
└── statistic_analysis/
```

These folders are mainly used for report writing and website construction.

* `outputs/exploratory_data_analysis/`
  Stores saved outputs from the exploratory data analysis notebook.
* `outputs/intake_form_visualization/`
  Stores simple participant background visualizations based on the intake form, mainly used for quick browsing and website display.
* `outputs/statistic_analysis/`
  Stores saved outputs from the statistical analysis notebook.

## 3) Software Dependencies

You can reproduce the main results using Python only.

### 3.1 Python Environment (required)

* Python ≥ 3.10
* JupyterLab or Jupyter Notebook
* NumPy
* Pandas
* Matplotlib
* Statsmodels

**Install with pip**

```bash
python -m pip install --upgrade pip
pip install jupyter numpy pandas matplotlib statsmodels
```

**Or with Conda**

```bash
conda create -n prob-teaching python=3.11 -y
conda activate prob-teaching
conda install -c conda-forge jupyter numpy pandas matplotlib statsmodels -y
```

## 4) How to Reproduce Results

### Step 1 — Exploratory Data Analysis

Open `exploratory_data_analysis.ipynb` and run **top-to-bottom**.
This notebook performs sanity checks and produces EDA outputs saved under `outputs/exploratory_data_analysis/`.

### Step 2 — Intake Form Visualization

Open `intake_form_visualization.ipynb` and run **top-to-bottom**.
This notebook generates simple visual summaries of participants’ backgrounds based on intake form responses and saves outputs under `outputs/intake_form_visualization/`.

### Step 3 — Statistical Analysis

Open `statistic_analysis.ipynb` and run **top-to-bottom**.
This notebook fits models for hypothesis testing and saves analysis outputs under `outputs/statistic_analysis/`.
