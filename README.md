### 2.1 `data/raw/` (do not edit)

`raw/` contains the original exports for **all participants**, including those who did not finish.

Files:

* `basic.csv`
  Group/section assignment and basic grouping information.
* `final_assessment.csv`
  Detailed final assessment records, including per-subquestion scores and confidence levels.
* `grades.csv`
  Aggregated scores for coding/handwritten/final.
* `intake_form.csv`
  Intake form responses (e.g., courses taken, background info).

### 2.2 `data/cleaned/` (generated; completed-only)

`cleaned/` contains cleaned versions of the same four files **filtered to only include students who completed the required tasks**, plus an additional status file.

Files:

* `basic.csv`, `final_assessment.csv`, `grades.csv`, `intake_form.csv`
  Same schemas as `raw/`, but restricted to вЂњcompletedвЂќ participants (see `status.csv`).
* `status.csv`
  Records each participantвЂ™s section/group and completion status (used to filter вЂњcompletedвЂќ participants).

### 2.3 Confidence-adjusted final score (what it means)

In `cleand/grades.csv`, the confidence-adjusted final score is computed using per-subquestion information from `final_assessment.csv`.

Conceptually:

* For each subquestion ( q ): `adjusted_q = score_q * (confidence_q - 1)` (let the confidence_q' start 0)
* The overall adjusted score is then aggregated across subquestions (weighted mean as implemented).

> The exact aggregation rule used is implemented in the pipeline code/notebook and should be treated as the source of truth.

### 2.4 `data/eda/` and `data/statstical analysis/`

These folders store **saved outputs** (figures/tables) generated by notebooks, used for writing the report and building web/GitHub Pages visuals.
